Combined Entrypoint

The runnable combined entrypoint is schematron/thermml.sch. That file mirrors the currently published rule families and is the Schematron layer used by the repository-level validate_one.py command.

Layout Note

Each rule family currently has two Schematron files with different roles.

  • module.sch is the shared include fragment that contains the actual rule-family body.
  • rule.sch is a thin standalone wrapper that includes module.sch so the family can still be executed by itself.

This split exists because the current lxml.isoschematron workflow supports include-based composition cleanly when the included file is a Schematron fragment such as a top-level pattern, not a full independent schema used as an include target.

Fixture Convention

  • valid.xml: passes without errors
  • invalid.xml: fails at least one assert
  • warning.xml: passes but emits one or more report diagnostics when the rule family uses warnings
